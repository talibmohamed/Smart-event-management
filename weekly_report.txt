Project summary:
Quickseat is a web application for creating, browsing, booking, and managing university events. The platform supports attendees, organizers, and admins using a Node.js/Express backend, Prisma ORM, PostgreSQL/Supabase, JWT authentication, Stripe payments, Supabase Storage, Resend email delivery, Socket.IO notifications, Redis-ready workers, and a React frontend built with Vite, Tailwind CSS, and HeroUI.

- Selected the main technical stack: Node.js, Express.js, Prisma, PostgreSQL/Supabase, JWT, React, Vite, Tailwind CSS, and HeroUI.
- Designed the core data model around users, events, bookings, and feedback.
- Defined user roles and permissions: attendee, organizer, and admin.
- Planned the MVC-style backend structure and frontend page structure.
- Prepared the main authentication and event-management flows before implementation.

Challenges:
- The main challenge was keeping the architecture simple enough for a university project while still supporting real features such as authentication, payments, images, and maps.
- Several future integrations needed early planning, especially Stripe payments, Supabase storage, and geocoded event locations.
