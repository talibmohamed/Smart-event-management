Weekly Report - Smart Event Management

Project summary:
Smart Event Management is a web application for creating, browsing, booking, and managing university events. The platform supports attendees, organizers, and admins using a Node.js/Express backend, Prisma ORM, PostgreSQL/Supabase, JWT authentication, and a React frontend.

Week 1 (2026-03-23 to 2026-03-29)

Completed:
- Defined the project scope: organizers manage events, attendees book events, and admins supervise the platform.
- Selected the main technical stack: Node.js, Express.js, Prisma, PostgreSQL/Supabase, JWT, React, Vite, Tailwind CSS, and HeroUI.
- Designed the core data model around users, events, bookings, and feedback.
- Defined user roles and permissions: attendee, organizer, and admin.
- Planned the MVC-style backend structure and frontend page structure.
- Prepared the main authentication and event-management flows before implementation.

Challenges:
- The main challenge was keeping the architecture simple enough for a university project while still supporting real features such as authentication, payments, images, and maps.
- Several future integrations needed early planning, especially Stripe payments, Supabase storage, and geocoded event locations.

Next steps from Week 1:
- Build the frontend project foundation.
- Implement routing, layout, authentication context, and protected routes.
- Start connecting frontend pages to backend endpoints.

Week 3 (2026-04-06 to 2026-04-13)

Completed - Backend and database:
- Integrated Prisma ORM and refactored backend models for User, Event, Booking, and Feedback.
- Connected the backend to PostgreSQL/Supabase.

- Updated frontend event forms, cards, details, dashboard, filters, and map behavior to use address, city, latitude, and longitude.

Completed - Images:
- Added Supabase Storage support for event cover images.
- Added backend image upload handling and validation for JPEG, PNG, and WebP files under 5MB.
- Updated frontend create/edit event forms with cover image upload, preview, replace, and remove behavior.
- Displayed event images on event cards, event detail pages, dashboard cards, and booking cards.

Completed - Bookings and payments:
- Built the My Bookings page.
- Added booking cancellation from the frontend.
- Integrated Stripe Checkout for paid event bookings.
- Implemented Stripe webhook-based booking confirmation on the backend.
- Added frontend payment status polling after Stripe redirect.
- Added support for pending payment, confirmed, cancelled, failed, expired, and cancelled checkout states.
- Added retry payment flow for existing pending bookings.
- Updated event availability UI using confirmed bookings, remaining seats, and full-event status.

Challenges:
- Stripe Checkout required careful separation between frontend redirect UX and backend webhook confirmation.
- Pending paid bookings must not reserve seats, so availability needed to depend only on confirmed bookings.
- Supabase image upload required coordination between backend storage setup and frontend multipart form submission.
- Map behavior needed refinement so zooming prioritizes events in the visible area without hiding the rest of the list.
